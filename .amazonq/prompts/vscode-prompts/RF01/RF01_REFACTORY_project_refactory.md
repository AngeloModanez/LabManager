Quero que você analise **todo o projeto**: backend, frontend e mobile.  
Identifique problemas, código inútil, arquivos desnecessários e más práticas.

Tarefas:

1. Analise o backend:
   - Verifique se há rotas, serviços, validações ou middlewares desnecessários.
   - Veja se o uso de PATCH é realmente necessário; caso não seja, remova e padronize para PUT.
   - Organize as pastas e módulos.
   - Aperfeiçoe controladores, serviços e models.
   - Padronize mensagens de erro, tratamento de exceções e respostas.
   - Melhore a estrutura geral e deixe o código mais limpo.

2. Analise o frontend:
   - Verifique inconsistências, componentes duplicados, lógica repetida ou código morto.
   - Padronize tudo com base nos componentes principais.
   - Reorganize a estrutura e deixe o código mais simples e coerente.

3. Analise o mobile:
   - Remova telas, arquivos ou componentes inúteis.
   - Padronize as telas seguindo o mesmo estilo do frontend.
   - Ajuste os serviços e chamadas para seguir os mesmos padrões do backend.
   - Deixe as telas, validações e feedbacks coerentes com o frontend.

4. Refatoração geral:
   - Melhore nomes de arquivos, pastas, funções e variáveis.
   - Sugira e aplique boas práticas de organização.
   - Deixe o projeto mais consistente, limpo, padronizado e fácil de manter.

Entregue:
- Lista do que foi removido ou reorganizado.
- Estrutura final das pastas.
- Explicação breve das melhorias.
- Código atualizado e refatorado para backend, frontend e mobile.
- Não tire e não mude o nome das pastas do backend, frontend e mobile.
- Deixe o backend, frontend e mobile dentro da pasta /projeto/ .
